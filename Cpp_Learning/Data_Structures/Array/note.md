# Array

## Array的定义

* 在 C++ 标准中，数组的大小必须是一个常量表达式（如 10 或 const int N = 10）。编译器在编译代码时，就已经计算好了这个函数需要多大的栈空间。

* 字符数组定义时要预留出'\0'的空间

* 不允许拷贝和赋值

* 理解复杂的数组声明时记住优先级。例如：int *(*matirx)[5] 这是一个指向5个指针数组的指针

## 数组下标越界
* 数组下标计数是从0开始的，当我们查找或者修改超出数组长度的数据时，就会有栈污染，此时就会改变数组相邻数据，导致错误。

## 指针与数组

* 当我们使用一个这样的指针：string *p1 = nums,这个指针的地址等价于nums[0].即数组的中中指针就是数组第一个元素的指针。

## 多维数组

* 多维数组在实际存贮中并非是网格式存储，它在在内存上线性存储

* 使用多维数组的时候要注意栈溢出的风险，大型数据要使用vector存储数据，数组扮演遥控器。
## Array && Vector

### 数组每次写之前 两者比较还需要设定大小，那我直接用vector不就可以了吗？为什么还要用数组呢？

* array存放在栈上，vector实际存储数据在堆上，所以array的调用速度快，缓存友好

* vector实现无限长度，内部v有三个维护指针，访问速度慢

* vector虽然是动态，但是并非无限，当vector超过自身容量后将触发搬家，代价很沉重
